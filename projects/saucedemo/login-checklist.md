# Чек-лист тестирования страницы логина (SauceDemo)

## Предварительные условия
- Открыт браузер Chrome (последняя версия).
- Перейти на https://www.saucedemo.com/

## Проверки пользовательского интерфейса (UI)
- [ ] Поле "Username" отображается и имеет плейсхолдер "Username"
- [ ] Поле "Password" отображается и имеет плейсхолдер "Password"
- [ ] Кнопка "Login" отображается и кликабельна
- [ ] Логотип "Swag Labs" отображается вверху страницы
- [ ] Кнопка "Login" меняет цвет при наведении (hover)
- [ ] Поля ввода имеют рамку при фокусе

## Функциональные проверки (позитивные)
- [ ] Ввод валидного логина "standard_user" и пароля "secret_sauce" → успешный вход (редирект на /inventory.html)
- [ ] После успешного входа в URL отображается /inventory.html
- [ ] После успешного входа отображается страница со списком товаров

## Функциональные проверки (негативные)
- [ ] Пустые поля → кнопка Login неактивна или выдаёт ошибку (фактически: ошибка "Username is required")
- [ ] Заполнен только username, пустой пароль → ошибка "Password is required"
- [ ] Заполнен только пароль, пустой username → ошибка "Username is required"
- [ ] Неверный логин (например, "invalid_user") и верный пароль → ошибка "Username and password do not match any user"
- [ ] Верный логин и неверный пароль → ошибка "Username and password do not match any user"
- [ ] Спецсимволы в логине/пароле (например, "!@#$%") → обработка ошибки (сервер должен вернуть корректный ответ)

## Дополнительные проверки
- [ ] Поле пароля скрывает вводимые символы (type="password")
- [ ] Нет возможности сохранить пароль (браузер не предлагает сохранить, если это не настроено)
- [ ] После неудачной попытки входа поля не сбрасываются (username остаётся, пароль очищается — проверить фактическое поведение)
- [ ] Сообщение об ошибке исчезает при повторном вводе данных

## Данные для тестирования
- **Валидные пользователи:** standard_user, locked_out_user, problem_user, performance_glitch_user, error_user, visual_user
- **Пароль для всех:** secret_sauce
