# Чек-лист тестирования API DummyJSON (posts)

## Общие проверки
- [ ] Базовый URL доступен (GET https://dummyjson.com)

## GET /posts
- [ ] Статус 200 OK
- [ ] В ответе есть массив posts
- [ ] Массив содержит поля id, title, body, userId, tags, reactions
- [ ] Работает пагинация (параметры limit, skip)

## GET /posts/{id}
- [ ] Для существующего ID (например, 1) возвращается 200 и объект с правильным id
- [ ] Для несуществующего ID (например, 9999) возвращается 404 с сообщением об ошибке

## POST /posts/add
- [ ] С валидными данными возвращает 201/200 и объект с новым id
- [ ] Если не передать обязательное поле (title, body, userId) — статус 400
- [ ] Если передать невалидный JSON — статус 400
- [ ] В ответе при создании приходит поле id, которое можно сохранить в переменную

## PUT /posts/{id}
- [ ] Обновление существующего поста (например, /posts/1) возвращает 200 и обновлённый объект
- [ ] При обновлении несуществующего поста — 404

## DELETE /posts/{id}
- [ ] Удаление существующего поста возвращает 200 (имитация)
- [ ] При удалении несуществующего поста — 404

## Негативные тесты
- [ ] Отправка запросов с неверным методом (например, POST на GET-эндпоинт)
- [ ] Передача некорректных типов данных (строка вместо числа)
- [ ] Слишком длинные значения полей
